The aim of the project is to **design and develop a mobile application for
diet and nutrition**.
The students should conduct a **market and literature research** in order to
**explore different types of recommendation apps**, as well as the **features
offered by the recommendation apps/systems in nutrition and dietetics**.
Based on these analyses they need to design and develop a mobile
application that can be used for **dietary recommendation**.
The aim of the app is to help users to develop a **healthy diet** and **track their
progress** using the app. Some of the main functionalities should include:
- allowing user to define the goal (calorie limit per day),
- generate meal recommendations based on the user's calorie goal,
- allowing user to insert the calorie intake per meal,
- using online API to obtain recipes for the recommendations,
- allowing user to insert recipes,
- user diary, daily/monthly calorie intake (diet) reports.
---
Suggested Key Features for Diet and Nutrition (Mobile Application):
1. User Profile Management - The system shall allow users to create and manage a personal profile containing basic demographic and dietary-related information.
2. Dietary Goal Definition - The system shall allow users to define daily dietary goals, including a target daily calorie intake.
3. Meal Recommendation Generation - The system shall generate personalized meal recommendations based on the user's defined dietary goals and calorie limits.
4. Meal Categorization - The system shall classify recommended meals into standard meal categories, including breakfast, lunch, dinner, and snacks.
5. Calories Intake Logging - The system shall allow users to manually record calorie intake for each meal consumed during the day.
6. Recipe Retrieval via External API - The system shall retrieve meal recipes and associated nutritional information from external online recipe APIs.
7. Custom Recipe Entry - The system shall allow users to manually add, edit, and store their own recipes, including ingredient lists and estimated calorie values.
8. Diet Diary Management - The system shall maintain a diet diary that records user's daily calorie intake over time.
9.  Dietary Reporting - The system shall generate daily and monthly dietary reports summarizing total calorie intake and consumption trends.
10. Data Visualization - The system shall present dietary reports using graphical visualizations such as charts or graphs to support user understanding of intake patterns.

Advanced Functionalities:
1. Image-Based Meal Recognition - The system shall allow users to capture images of meals and automatically identify food items using image recognition techniques.
2. Automated Calorie Estimation from Images - The system shall estimate the calorie content of recognized food items based on image analysis and nutritional databases.
3. Diet Progress Tracking - The system shall track the user's dietary progress by comparing actual calorie intake against defined dietary goals over time.
4. Progress Feedback Generation - The system shall feedback to users regarding their progress toward dietary goals (e.g., on-track, exceeding limit, or under target).
5. Dietary Consultation via Chat Interface - The system shall provide a chat-based interface through which users can seek dietary advice from a nutrition expert or an AI-based dietary assistant.
6. Recommendation Adaptation - The system shall adapt future meal recommendations based on the user's historical dietary behaviour and progress trends.
7. Dietary Preference and Restrictions Support  - The system shall allow users to specify dietary preferences or restrictions, such as vegetarian, vegan, halal, or food allergies.

Note: These functionalities are suggested. The team should try to implement as much as possible. Not all the listed functionalities must be implemented. The team need to plan according to the man-power resources available. However, the grade awarded will be based on the functionalities implemented.
